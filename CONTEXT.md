# git-wt ‚Äî Context

`git-wt` is a git extension that makes worktrees agent-friendly: `git wt new <branch>` creates a worktree pre-provisioned with the ignored state a fresh checkout is normally missing, under one predictable naming convention.

## Glossary

**Main Checkout** ‚Äî the original clone of the repository (the worktree that owns `.git` as a directory). All provisioning sources come from here.

**Worktree** ‚Äî a linked `git worktree` created by `git wt new`, always placed by the repo's Placement.

**Placement** ‚Äî where worktrees live, set by the `worktrees` key in `.worktree.toml`: a preset (`sibling`, `inside`, `home`) or a path template with `{repo}`/`{branch}`. The default is the Sibling Convention. Slashes in branch names become dashes (`feature/auth` ‚Üí `feature-auth`).

**Sibling Convention** ‚Äî the default Placement: worktrees live in `../<repo>.worktrees/<branch-sanitized>` next to the Main Checkout.

**Cloned State** ‚Äî mutable, per-worktree ignored files (e.g. `node_modules`, `.turbo`) copied from the Main Checkout via filesystem copy-on-write (APFS `clonefile`). The clone costs ~zero disk/time; a file becomes a real independent copy only when written. Writes in a worktree never affect the Main Checkout.

**Shared State** ‚Äî singleton ignored files where write-through is *desired* (e.g. `.env`, a local dev database). Symlinked back to the Main Checkout; all worktrees see and mutate the same file.

**Skipped State** ‚Äî ignored paths intentionally not provisioned (e.g. `dist`); regenerated by builds in the worktree.

**Classification** ‚Äî the mapping of ignored paths to Cloned / Shared / Skipped. Built-in defaults, overridden per path by the repo's `.worktree.toml`.

**Setup Command** ‚Äî command always run after provisioning (default: detected package-manager install, e.g. `bun install`). Corrects staleness when the branch's lockfile diverges from the Main Checkout's; a no-op when they match. `git wt new` returns only when the worktree is coherent.

## Decisions

Cloned State uses CoW clones rather than symlinks or hardlinks: symlinks would let a worktree's writes leak back into the Main Checkout, and hardlinks share content the moment either side edits in place. CoW clones keep writes private to each worktree at near-zero disk/time cost on APFS.

A worktree is *managed* when its path round-trips through the Placement template (`WorktreePath(branch) == path`), not by directory-prefix matching ‚Äî templates like `~/wt/{repo}-{branch}` have no clean container directory. Consequence: changing `worktrees` orphans previously created worktrees from `ls`/`rm`/`prune`; they remain plain git worktrees.

The `inside` Placement ignores its container via `.git/info/exclude` (written lazily by `new`, eagerly by `init`) rather than a committed `.gitignore` change: per-clone, nothing to commit, and teammates' clones get it automatically the first time they run `git wt new`.
